# Спецификация: по-раздельная модель отчётов (sections-as-base)

> Статус: **ЧЕРНОВИК НА СОГЛАСОВАНИЕ** (реализация не начата).
> Запрошено Пользователем 2026-05-17: «сами разделы настраиваются в
> базовом шаблоне; шапка/подвал не "первая/остальные", а по разделам —
> сколько разделов, столько колонтитулов».

## 1. Идея

- **Базовый шаблон** владеет раскладкой страницы: упорядоченный
  массив `sections[]`. Каждый раздел = страничная настройка со
  СВОЕЙ геометрией и СВОИМ колонтитулом (шапка+подвал).
- **Шаблон документа** ссылается на базу (`baseTemplateId`). Сам он
  НЕ задаёт геометрию/колонтитулы. Документ только:
  - наполняет тело (flow + структурные блоки),
  - блоком «разрыв раздела» переключается на СЛЕДУЮЩИЙ раздел базы.
- Модель колонтитулов `{firstPage, otherPages}` упраздняется. Кейс
  «другая первая страница» = раздел 0 базы (своя/пустая шапка),
  раздел 1 = тело; документ ставит разрыв после титула.

## 2. Структуры данных (template.js)

```js
tpl.sections = [{
  id: 's1',
  name: 'Основная',
  page: { format, orientation, width, height,
          margins:{ top,right,bottom,left } },
  header: { enabled, height, width:'print'|'page'|'bleed', bleed,
            valign:'top'|'middle'|'bottom',
            blocks:[{ type:'paragraph', style:'caption', align, text }] },
  footer: { ...та же форма... },
  repeat: true            // колонтитул повторяется на всех стр. раздела
}]
```

- База: ≥1 раздел. Раздел 0 = основной; доп. разделы (напр.
  альбомное приложение) добавляются в редакторе базы.
- `tpl.header` / `tpl.footer` (firstPage/otherPages) — **deprecated**,
  только как вход для миграции; после миграции очищаются.
- Документ хранит `flow[]` + `baseTemplateId`; геометрии/колонтитулов
  НЕ хранит.

## 3. Тело документа → разделы базы

- Тело раскладывается по разделам последовательно.
- Блок `sectionBreak` в потоке документа = «дальше использовать
  СЛЕДУЮЩИЙ раздел базы» (клампится последним). Документ на разрыве
  НЕ задаёт геометрию — только точку переключения.
- Документ не может переопределять геометрию/колонтитулы (в редакторе
  уже ограничено, v0.60.700; здесь финализируется).

## 4. Миграция (одноразовая, идемпотентная)

`migrateSectionsModel(tpl)`:
- Если `tpl.sections` уже непустой массив → нормализовать форму,
  выйти.
- Иначе синтез из legacy с НУЛЕВОЙ регрессией:
  - `overlays`-колонтитулы → существующий `applyColontitleBands`.
  - 1 колонтитул (firstPage == otherPages) → ОДИН раздел
    (`tpl.page` + otherPages-колонтитул) → вывод идентичен текущему.
  - firstPage ≠ otherPages → ДВА раздела: раздел0 (titre, repeat:false,
    firstPage-колонтитул, геом `tpl.firstPage.page||tpl.page`) +
    раздел1 (repeat:true, otherPages-колонтитул, `tpl.page`).
  - legacy `sectionBreak` с geom-override → доп. разделы базы (геом
    из разрыва), разрыв документа становится «переключателем».
  - Очистить `tpl.header`/`tpl.footer` после построения `sections`.

## 5. Рендер (preview / export-pdf / export-docx)

- `flowSegments(tpl)` уже режет тело по `sectionBreak`. Изменение:
  сегмент N берёт `base.sections[min(N, len-1)]` (база резолвится по
  `baseTemplateId`) для геометрии+колонтитула вместо first/other.
- `buildPageShell` / `drawHeaderFooter`: header/footer/valign/width
  из раздела сегмента. `colontitleBox` без изменений.
- Колонтитул раздела повторяется на всех его страницах. Спец-логика
  `isFirst` убирается (титул = раздел 0, если автор его создал).
- DOCX: одна Word-секция на раздел базы (мультисекции уже есть).
- Fallback: если `sections` нет — старый путь (на время поэтапного
  внедрения), нулевая регрессия.

## 6. Редактор (UX)

- **Базовый шаблон** — вкладка **«Разделы»**: список разделов;
  карточка раздела: имя, страница (формат/ориентация/поля), и его
  **Шапка**/**Подвал** (вкл, высота, ширина-режим, bleed, valign,
  многострочный текст, h-align). «+ Добавить раздел», reorder,
  удалить. Текущие base-вкладки «Лист»/«Колонтитулы» сворачиваются
  в посекционное редактирование здесь.
- **Шаблон документа** — «Основа» (выбор базы, обязателен),
  «Структура» (тело + sectionBreak = «перейти к следующему разделу
  базы», показывает в какой раздел базы попадает часть, геометрия
  read-only), «Слой». Колонтитулов/полей/стилей в документе нет.
- valign сохраняется нативно (на `section.header/footer`, без
  roundtrip через legacy-overlay) — чинит дефект persist из v0.60.700.

   модель «спящая»). Verify: миграция идемпотентна, legacy → sections
   эквивалентны.
2. Рендер читает `sections` (preview→pdf→docx). Verify: попиксельная
   эквивалентность legacy-шаблонов; новые многосекционные — корректны.
3. Редактор базы: вкладка «Разделы» с посекционным колонтитулом.
4. Документ: «Структура»/«Разделы» read-only карта тело↔раздел базы.
5. Seed: встроенные базы → `sections`; `BUILTIN_VERSION++`.

Откат: per-phase; модель «спящая» до этапа 2; рендер за fallback на
legacy при отсутствии `sections`.
