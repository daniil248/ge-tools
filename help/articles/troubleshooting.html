🛠 Типичные проблемы и решения

Если что-то пошло не так — сначала сделайте бэкап (кнопка «💾 Бэкап» на /projects/), и только потом пробуйте чинить. Это страховка на случай если ремонт усугубит проблему. См. «💾 Резервное копирование».

📁 Проекты «пустые», 0 узлов / стоек / связей

Симптом: в /projects/ карточка проекта показывает «0 узлов в схеме · 0 стоек · 0 СКС-связей · 0 IT-устройств».

Причины:

  • Проект-контейнер был автоматически создан из cloud-схемы, но cloud-схема ещё не загружена в локальный кеш.
  • Произошёл сбой UID-scope (см. ниже) и ваши данные хранятся под другим scope.
  • Контейнер действительно пуст — вы только что его создали.

Решение:

  1. Если в /projects/ есть раздел «Без проекта · N схем» — нажмите «🔧 Восстановить связи». Это привяжет orphan-схемы к контейнерам по совпадению имени.
  2. Откройте проект в Конструкторе схем («↑ Применить в Конструкторе») — облачная схема загрузится автоматически (если вы залогинены).
  3. Если ничего не помогло — восстановите из бэкапа.

👤 Пропали мои проекты / схемы / шкафы / пресеты

Симптом: В реестре пусто, хотя раньше там были данные.

Возможные причины:

  • UID-scope сбит. Часть данных хранится под Firebase UID. Если auth.js перезаписал UID на 'anonymous' (был баг до v0.59.851), per-user данные стали невидимыми. После v0.59.851 этого больше не происходит.
  • Очистка cookies/cache в браузере. Это уничтожает LocalStorage. Если вы делали Ctrl+Shift+Del и галку «Cookies and other site data» — данные физически удалены.
  • Другой профиль/инкогнито/браузер. Работаете в другом контексте.
  • Orphan project-data. Метаданные проекта удалились, но содержимое (POR-объекты, схема) живёт в raschet.project.<pid>.*. См. ниже.

Что проверить:

  1. Откройте F12 → Application → LocalStorage → найдите ключ raschet.projects.v1. Если он есть и содержит ваши проекты — данные на месте, проблема в отображении (попробуйте «🔧 Восстановить связи»).
  2. Если ключа нет совсем — данные удалены из браузера. Восстановите из бэкапа (см. «💾 Резервное копирование»).
  3. Откройте /dev/por-playground.html — там список «Найдено в LocalStorage» покажет ВСЕ pid-ы с POR-данными. Если есть pid-ы, которых нет в /projects/ — это orphan, восстанавливаются кнопкой «🔧 Восстановить связи».

🔗 Схемы в «Без проекта», не привязаны к контейнерам

Симптом: На /projects/ есть раздел «Без проекта · N схем» с карточками схем. Контейнеры с такими же именами могут быть рядом или отсутствовать.

Решение: Кнопка «🔧 Восстановить связи» на /projects/ запускает migrateOrphanSchemes(), который:

  1. Сканирует LocalStorage на ключи raschet.project.<pid>.* и находит pid-ы без metadata-записи. Создаёт записи (с теми же id) — без потери привязки к данным.
  2. Для каждой orphan-схемы (lp_*) пытается найти контейнер с тем же именем (case-insensitive, trim+space-collapse). Если найден — линкует. Иначе создаёт контейнер.

💧 Список не отсортирован по обозначению

Симптом: SR05, SR08, CR01, MR1, SR01... вместо CR01, MR1, SR01, SR02, SR03, SR04, SR05, SR08.

Причина: старая версия. С v0.59.844+ применяется natural-sort везде где есть теги.

Решение: обновить страницу (Ctrl+F5).

🚫 «+ в стойку» всегда ставит на U=1, накладываются

Был баг до v0.59.846 (r.occupied undefined → NaN → fallback 1). Исправлено.

🎨 Контейнер потребителей рендерится чёрным

Был баг до v0.59.824 (CSS class). Исправлено — контейнер наследует стиль consumer.

📏 Линии связей не кликаются в SCS-design

Был баг до v0.59.849 (CSS pointer-events). Исправлено — клик по линии выделяет её, Shift+клик — добавляет к множественному выделению.

⚙ Шестерёнка ⚙ показывает только справочник напряжений

С v0.59.855 в модалке настроек добавлена секция «💾 Резервное копирование». Если её не видно — обновите страницу (Ctrl+F5).

🌐 Авторизация: «Войти» постоянно появляется в каждом модуле

До v0.59.834 модули scs-config / scs-design / mv-config / mdc-config / catalog / rack-config / logistics не подключали Firebase auth. С v0.59.834 они синхронизированы — единая Google-сессия видна везде.

Если в каком-то модуле «Войти» всё равно появляется при наличии сессии в других — это значит ваш браузер блокирует cross-tab синхронизацию Firebase (Privacy Sandbox / strict mode). Попробуйте обновить страницу.

☁ Облачные схемы не подгружаются в карточку проекта

Карточка проекта /projects/project.html?project=<pid> показывает счётчики только для LS-данных. Если cloud-схема существует, но в LS её ещё нет — счётчики 0.

Решение: нажмите «↑ Применить в Конструкторе» на карточке проекта — Конструктор сам загрузит cloud-схему.

💀 Восстановление невозможно (данные ушли совсем)

Если LocalStorage был очищен (cookies wipe / новая инсталляция Chrome / другая машина), и облако пусто или вы не залогинены — единственный путь восстановления это раньше сделанный бэкап-файл.

Поэтому настройте авто-бэкап один раз и забудьте. См. «💾 Резервное копирование».

🆘 Если совсем застряли

  1. Сделайте ручной бэкап (даже сломанного состояния — полезно для диагностики).
  2. Откройте F12 → Console → пришлите скриншот ошибок (красные строки).
  3. Откройте /dev/por-playground.html → пришлите список «Найдено в LocalStorage».
  4. Опишите: что нажимали, какой версии, что ожидали, что получили.
