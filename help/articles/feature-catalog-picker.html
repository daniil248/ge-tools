Зачем

На предпроектной стадии иногда нужно сразу выбирать конкретные модели изделий — например, заказчик заранее хочет Schneider Galaxy VS 100 кВА, а не «100 кВА абстракцию». Привязка к каталогу позволяет:

  • Авто-копировать паспортные параметры (kW, kVA, размеры, cosφ)
  • Передавать modelRef в schematic при handoff'е
  • Считать стоимость concept-уровня через цены каталога

Как использовать

  1. В любой секции tech-workspace (стойка / ИБП / кондиционер / ТП / ДГУ / PDU) нажмите 📦 Привязать модель…
  2. Откроется модалка с поиском по производителю/модели.
  3. Кликните на нужную строку — параметры скопируются в карточку, кнопка станет зелёной с указанием модели.
  4. Чтобы изменить — кликните на ту же кнопку (теперь она с ✏ карандашом). Кнопка 🗑 Снять привязку в модалке отвязывает.

Каталог-источник

Все picker'ы используют общую библиотеку shared/element-library.js. Включает:

  • Builtin-элементы (зашиты в код, обновляются с релизами)
  • User-элементы (добавлены пользователем через модуль 📦 Каталог)
  • Override'ы builtin'ов (пользователь может переопределить параметры)

Хранение привязки

variant.concept.rackGroups[0].modelRef = {
  id: 'el-rack-eaton-rstck-42u',
  manufacturer: 'Eaton',
  model: 'RS Rack 42U 600×1200',
  kind: 'rack'
}

Привязка хранится по id — устойчива к переименованиям модели в каталоге.

Что попадает в handoff

При передаче в проектирование, modelRef передаётся в engine.scheme.v1 как поле узла:

node._fromTechWorkspace = true;
node._modelRefId = rackGroup.modelRef.id;
