Концепция проекта

Проект — это контейнер для всей документации одного объекта: схема электроснабжения, план зала СКС, реестр оборудования, шкафы, концепция технолога, отчёты.

Структура хранения

Создание проекта

  1. На главной странице (Hub) откройте карточку «Проекты».
  2. Кнопка ➕ Новый проект.
  3. Введите имя; проект создаётся с дефолтной (пустой) схемой.
  4. В карточке проекта виден список модулей, доступных для проекта (Технолог, Схема, СКС, МЦОД и др.).

Local / Online режимы

В шапке любого модуля есть переключатель режима хранения (см. статью). По умолчанию — auto:

  • Если Gmail-аккаунт авторизован → проекты в Firebase Firestore (совместная работа).
  • Иначе — LocalStorage браузера.

Принудительно можно переключить в local чтобы не тратить квоту Firebase, или в cloud чтобы использовать только облако.

⚠ Local-проекты с id lp_*, p_*, s_* не уходят в облако — для них кнопка sync ⬆ Local→Cloud копирует данные с новыми cloud-id.

Совместная работа

В Online-режиме проект можно «расшарить» с коллегами по Gmail. Они получают доступ через карточку проекта. Изменения синхронизируются live (Firestore listeners).
