💾 Резервное копирование и восстановление

Платформа Raschet хранит ВСЕ ваши данные (проекты, схемы, СКС, шкафы, каталоги, пресеты) в LocalStorage браузера + опционально в облаке Firebase. Это значит:

  • Очистка cookies/cache в браузере → данные ВНУТРИ браузера могут пропасть
  • Переход на другой компьютер / другой браузер → доступ только к облачным данным
  • Глюк миграции / ошибка платформы → возможна потеря данных

Поэтому обязательно настройте резервное копирование при первом использовании.

Способ 1: ручной бэкап (всегда работает)

  1. Откройте «📁 Проекты» (/projects/).
  2. В правой части шапки нажмите кнопку «💾 Бэкап».
  3. Браузер скачает файл raschet-backup-YYYY-MM-DD-HH-mm.json в стандартную папку загрузок.
  4. Скопируйте этот файл в надёжное место (Dropbox, Google Drive, корпоративный сервер, флешка).

Бэкап — это JSON-снимок ВСЕХ ключей LocalStorage (включая ключи всех модулей: scs-config, rack-config, ups-catalog и т.п.). Размер обычно 50–500 KB.

⏱ Делайте ручной бэкап после крупных изменений (импорт проекта, массовое наполнение шкафов, выгрузка из облака). И всегда — перед обновлением версии Raschet.

Способ 2: авто-бэкап в локальную папку (Chrome / Edge)

Один раз настраивается, дальше платформа сама пишет JSON-снимки в выбранную папку — например D:\Raschet-Backups\.

  1. Нажмите ⚙ в правом верхнем углу любой страницы Raschet.
  2. В разделе «💾 Резервное копирование» нажмите «📁 Выбрать папку для авто-бэкапа».
  3. Браузер спросит подтверждение разрешения на запись — выберите папку и подтвердите.
  4. Включите чекбокс «Включить авто-бэкап в выбранную папку».
  5. Задайте интервал (рекомендуется 60 минут).
  6. Опционально включите «Доп. бэкап при закрытии вкладки» (best-effort).

Платформа будет писать файлы вида raschet-backup-2026-04-30-15-42.json в выбранную папку каждый интервал. Не забывайте периодически копировать содержимое папки в надёжное место.

⚠ File System Access API работает только в Chrome 86+, Edge 86+, Opera 72+. В Safari и Firefox он не поддерживается — там доступен только Способ 1 (ручной бэкап).

Способ 3: облачное хранение (рекомендуется для команды)

Авторизуйтесь через Google (кнопка 👤 Войти в шапке). Данные проектов будут синхронизироваться с Firebase Firestore. Подробнее — статья «💾☁ Режим хранения: Локальный / Онлайн».

Важно: облако ≠ замена бэкапу. Делайте локальный бэкап даже если работаете в облачном режиме — на случай ошибки синхронизации или удаления данных.

📂 Восстановление из бэкапа

  1. Откройте «📁 Проекты».
  2. Нажмите «📂 Восстановить».
  3. Выберите ранее сохранённый JSON-файл.
  4. Появится диалог с двумя стратегиями:
    • MERGE (рекомендуется) — данные из бэкапа добавятся к текущим. Если ключ существует и в бэкапе, и сейчас — берётся значение из бэкапа.
    • REPLACE — сначала localStorage.clear(), потом записать всё из бэкапа. Полное замещение текущего состояния.
  5. После подтверждения страница перезагрузится, данные восстановлены.
💡 Если случайно перезаписали свой бэкап неудачным состоянием — восстанавливайте из предыдущего файла (он остался в папке Downloads).
