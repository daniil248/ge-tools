Зачем

Firebase Firestore (бесплатный тариф) имеет дневной лимит на запись. При активной работе (особенно с автосохранением) лимит можно исчерпать — сохранение начинает падать с resource-exhausted.

Решение — пользователь явно выбирает режим хранения:

  • 💾 Локально — все данные в LocalStorage браузера, без квот, без сети. Идеально для одиночной работы.
  • ☁ Онлайн — Firebase Firestore. Совместная работа, синхронизация между устройствами.
  • 🔄 Авто (default) — Cloud если Firebase авторизован, иначе Local.

Где переключается

В шапке любого модуля (рядом с шестерёнкой ⚙) — кнопка-badge:

  • 💾 Локально — текущий режим Local
  • ☁ Онлайн — Cloud
  • 🔒 рядом с режимом — означает что выбран override (не auto)

Click → модалка с тремя радио-кнопками + secciones синхронизации.

Синхронизация

При смене режима данные не переносятся автоматически. Используйте sync-кнопки:

  • ⬆ Local → Cloud — копирует все local-проекты в Firebase. Local-id (lp_*, p_*, s_*) получают новые cloud-id.
  • ⬇ Cloud → Local — копирует все cloud-проекты в LS как «снимок». Полезно перед длительной офлайн-работой.
⚠ Sync ручной (manual). Не автоматический. Это сохраняет контроль за квотой Firebase.

Технические детали

// js/projects.js
const STORAGE_MODE_KEY = 'raschet.storageMode.v1';
function getUserStorageMode() {
  return localStorage.getItem(STORAGE_MODE_KEY) || 'auto';
}
function getStorage() {
  const userMode = getUserStorageMode();
  if (userMode === 'local') return Local;
  if (window.Auth?.isFirebaseReady) return Fs;
  return Local;
}

Глобальный window.Storage.userMode / setUserMode(m) + event raschet:storage-mode-changed для обновления UI.

Когда что выбирать

СценарийРекомендуемый режим
Одиночная работа, много правок💾 Локально (нет квот)
Команда из 2-3 проектировщиков☁ Онлайн (live-sync через Firestore)
Дорога / поездка без интернета💾 Локально + перед уходом ⬇ Cloud→Local
Quick-эксперимент, не привязанный к проекту💾 Локально
