Что это

Handoff — передача активного варианта концепции в детальное проектирование. Создаёт engine.scheme.v1 проекта (главную схему) с предзаполненными узлами по данным концепции. Варианту присваивается read-only — повторно его редактировать нельзя.

Как использовать

  1. В tech-workspace откройте основной вариант (или любой выбранный).
  2. Кнопка 📤 Передать в проектирование в левой панели (зелёная).
  3. Появится диалог подтверждения с описанием того, что будет создано.
  4. Если в проекте уже есть схема — второе подтверждение «Заменить?». Внимание — старая схема будет потеряна.
  5. После handoff — предложение «Открыть Конструктор?» с переходом на ../index.html.

Что создаётся в схеме

Узлы:

  • source: TP (если feed.tp.needed) или Utility («Городская сеть»)
  • generator: ДГУ (если feed.dgu.needed) с capacityKw + backupMode
  • panel: ГРЩ — главный распределительный щит, inputs=2, outputs = upsSystems.length + coolingUnits.length
  • ups: по одному узлу на каждую upsSystem (тэг ИБП или ИБПК для climate)
  • consumer (rack): по одному на rackGroup, count = group.count, demandKw = group.kwPerRack
  • consumer (outdoor_unit): по одному на coolingUnit

Метаданные узлов:

node._fromTechWorkspace = true;     ← маркер происхождения
node._profile = 'it';                ← профиль из rackGroup (для GPU/Network)

Что НЕ создаётся (на MVP-уровне)

  • Связи (conns): ноды стоят на схеме, но не соединены. Электрик проводит линии вручную.
  • Автоматы и кабели: подбираются электриком после прокладки связей.
  • Расположение в схеме: базовый layout — источник слева, ГРЩ в центре, потребители справа. Электрик может перекомпоновать.
  • scs-design / scs-config: в MVP не затрагивается. scheme-rack-bridge при следующем входе в scs-config автоматически создаст виртуальные стойки из rack-узлов схемы.

Read-only после handoff

Variant получает readOnly: true и timestamp handoffAt. Все input/select в его модалке disabled. В сравнении (📊) появляется иконка 🔒.

⚠ Read-only необратим. Если нужно скорректировать — создайте копию варианта (📋 Дублировать) и работайте с ней.

Что дальше после handoff

  1. Открыть Конструктор схем — появится сгенерированная схема.
  2. Провести связи между узлами (drag из выходного порта одного → входной порт другого).
  3. Электрические параметры (cosφ, напряжение) уже заполнены из концепции.
  4. Электрик подбирает автоматы и кабели через инспектор узлов.
  5. СКС-инженер видит rack-стойки в Компоновщике (через scheme-rack-bridge).
