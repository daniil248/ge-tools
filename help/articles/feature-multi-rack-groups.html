Зачем

В реальном ЦОДе редко бывают «8 одинаковых стоек по 7 кВт». Обычно есть несколько групп с разными характеристиками:

  • Серверные (IT-rack, 600×1200 мм, 7 кВт каждая)
  • GPU-стойки (800×1200 мм, 12-25 кВт каждая)
  • Сетевые (network rack, 600×800 мм, 2-3 кВт)
  • Storage (горячая зона, 600×1200, 8 кВт)

Аналогично для ИБП — IT и климат могут питаться от разных систем (например, IT — от ИБП с N+1, климат — без резерва или 2N от другого ИБП).

Модель данных

concept = {
  rackGroups: [
    { id, name, count, kwPerRack, profile, widthMm, depthMm,
      modelRef,                                ← привязка к каталогу
      pdu: { kind, phases, ratingA,
             inputsPerRack, modelRef }         ← PDU настраивается per-rack-group
    },
    ...
  ],
  upsSystems: [
    { id, name,
      purpose: 'it' | 'cooling' | 'mixed',     ← разные цели
      count, ratedKva, redundancy,
      cosPhi, loadFactor,
      autonomyMin, batteryTech, modelRef
    },
    ...
  ],
  coolingUnits: [ … ],
  feed: { tp, dgu },
}

UI

В каждой секции есть кнопка ➕ Группа стоек / ➕ Система ИБП / ➕ Кондиционеры. Каждая карточка имеет:

  • Inline-редактирование имени (клик на название)
  • Поля параметров
  • Кнопку 📦 Привязать модель… (см. статью)
  • Кнопку × для удаления (минимум одна группа должна остаться)
  • Сводку справа: «8 × 7.0 кВт = 56.0 кВт» (динамически)

PDU per-rack-group

Раньше в концепции был один общий PDU-блок. По запросу пользователя — теперь PDU настраивается внутри каждой группы стоек: серверные могут иметь Metered 32А ×2 ввода, а GPU — Switched 63А ×4.

ИБП purpose: IT vs Cooling

Для каждой системы ИБП можно выбрать:

PurposeЧто считается
⚡ IT-нагрузкаПитает стойки. Должен покрывать Σ rackGroups × profile≠network.
❄ КлиматПитает кондиционеры. Может быть N+1 от другого ИБП.
🔄 СмешанноеСчитается и в IT, и в climate (для ИБП-резерва).

В summary секции «ИБП» показано отдельно: ⚡ IT доступно vs ❄ климат доступно. С margin (запас) — зелёный если ≥ нужно, красный если меньше.

Расчёт IT-нагрузки

Σ IT = сумма по всем rackGroups, исключая profile === 'network' (network-стойки — отдельная нагрузка, не классическая IT).

itTotal = Σ (group.count × group.kwPerRack), where group.profile ≠ 'network'

Миграция со старой модели

В версиях ≤ v0.59.783 концепция имела одно поле concept.racks (объект). При первом открытии в v0.59.784+ происходит автоматическая миграция в массив concept.rackGroups с одним элементом. Старые данные сохраняются.

💡 Совет: чтобы получить разные мощности — создайте 2-3 группы. Например, "Серверные ×6 7кВт" + "GPU ×4 15кВт" + "Сетевые ×2 3кВт".
