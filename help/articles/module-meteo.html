Назначение

Метеоданные — stand-alone модуль для импорта климатических рядов (температура, влажность, ветер) с привязкой к локации (lat / lon) и периоду наблюдений. Активный датасет используется в:

  • Технологе ЦОД — расчёт PUE по доле FreeCool-часов (T < 14 °C).
  • ID-диаграмме (психрометрия) — расчётные параметры наружного воздуха.
  • Климат-модулях — подбор кондиционеров по ASHRAE TC 9.9 на основе Tmin/Tmax/T99%.

Источники данных (plugin-архитектура)

Каждый источник — отдельный модуль в meteo/sources/. Чтобы добавить новый — создаётся файл с register({ id, label, async createDataset(ctx) }) + одна строка import './<name>.js' в sources/index.js. UI-кнопка появится автоматически.

Open-Meteo REST

Бесплатный публичный API без авторизации (archive-api.open-meteo.com). Вы вводите широту, долготу, период (до года) — получаете почасовой ряд температуры (T2m), относительной влажности (RH), скорости ветра (Ff). Запрос обычно занимает ≤2 секунд.

rp5 CSV

На rp5.ru/rp5.kz: «Архив погоды → Скачать», получаете файл CSV (semicolon-separated, UTF-8). Парсер автоматически распознаёт столбцы: «Местное время в...», T (температура), U (влажность), Ff (ветер). Время в формате DD.MM.YYYY HH:MM.

UI

  • Левый sidebar: список загруженных датасетов, кнопки импорта (генерируются динамически из реестра источников).
  • Правая панель: KPI (T средн / min / max / 99% / FreeCool ч / Записей), гистограмма распределения T (бины по 1 °C, зелёная подложка для T < 14 °C — это зона прямого FreeCool), таблица сводки HDD / CDD / T 1%.

Активный датасет

В списке датасетов кнопка ⭐ помечает текущий датасет как «активный для проекта». Активный используется другими модулями через meteo/meteo-api.js::getActiveDataset(pid).

Хранилище

  • raschet.project.<pid>.meteo.datasets.v1 — массив датасетов
  • raschet.project.<pid>.meteo.activeId.v1 — id активного датасета
  • raschet.meteo.last-loc.v1 — последняя введённая локация (для удобства)

Расширение

Чтобы добавить новый источник (NOAA, Росгидромет, Meteostat, Excel-импорт и т.п.), создайте файл meteo/sources/<name>.js по образцу:

import { register } from './registry.js';

register({
  id: 'noaa',
  label: '🌎 NOAA Climate Data',
  description: '...',
  async createDataset(ctx) {
    const { computeStats, modalOpen, toast } = ctx.util;
    return modalOpen('<h3>NOAA</h3>', '...', async () => {
      // выполнить fetch / парсинг
      const hourly = [...]; // [{ t: ISO, T: °C, RH, wind }, ...]
      return { name, source: 'noaa', lat, lon, locationName,
               dateFrom, dateTo, hourly, stats: computeStats(hourly) };
    });
  },
});

И добавьте import './noaa.js' в meteo/sources/index.js. Кнопка появится в UI автоматически.
