Назначение

Проектирование СКС — модуль для СКС-инженера: межшкафные связи, план зала, кабельный журнал.

Источники данных

  • Стойки берутся из engine-схемы (consumer-узлы со subtype=rack) через shared/scheme-rack-bridge.js.
  • Группы стоек (shell-container) автоматически разворачиваются: shell ×N → N виртуальных стоек с уникальными tag.
  • Для concept-групп с populated linkedAliases shell не создаёт виртуалов (slots уже linked к реальным узлам).

Возможности

  • План зала с drag-drop стоек
  • Кабельные трассы между шкафами
  • Кабельный журнал (марка, длина, тип, концы A→B)
  • Подпроекты СКС внутри проекта (несколько серверных в одном объекте)

Связи

  • scs-config — содержимое каждой стойки (юниты, оборудование)
  • schematic — общая электрическая схема, откуда стойки
  • shell-container — модель групп
