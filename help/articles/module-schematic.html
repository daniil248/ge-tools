Назначение

Конструктор схем — главный модуль платформы. Принципиальная схема электроснабжения с автоматическим расчётом нагрузок, подбором кабелей и автоматов.

Узлы и связи

Схема состоит из узлов (source / generator / panel / consumer / ups / channel / zone) и связей (cables, system-conns) между их портами.

Тип узлаЧто представляет
sourceИсточник питания: TP, utility, transformer
generatorДГУ (дизель-генератор)
panelРаспред. щит (ВРУ, АВР, ЩС)
upsИБП — моноблок или модульный
consumerНагрузка: стойка, рабочее место, освещение, вентиляция и т.п.
channelКабельный канал / трасса
zoneГруппировка узлов (helper-блок)

Расчёты

  • Нагрузки: Σ потребители × коэф. использования. Per-line и суммарно.
  • Токи КЗ: по схеме замещения (источник + трансформатор + ЛЭП + ...).
  • Падение напряжения (ΔU): накопленное от источника по линиям. Норма ±10% по ГОСТ 32144-2013.
  • Кабели: подбор сечения по IEC / ПУЭ с учётом температуры, способа прокладки, групп.
  • Автоматы: подбор номинала, проверка селективности с upstream.

Возможности

  • Группы потребителей (shell) — одна карточка для N одинаковых стоек
  • Пресеты карточек — фильтр полей по ролям (электрик / технолог / минимум)
  • Несколько страниц: главная схема + план зала (layout) + СКС-страницы
  • Режимы расчёта: РТМ 36.18.32.4-92, упрощённый, ручной
  • Экспорт: PNG, SVG, JSON-проект
  • Live-collab: presence, cursors, locks (Firestore mode)

Навигация по канвасу

  • ЛКМ drag по узлу — переместить
  • ЛКМ по порту → drag к другому порту — провести связь
  • ПКМ по линии — toggle линкмода (скрыть/показать ссылкой)
  • Колесо — масштаб
  • Ctrl+Z / Ctrl+Shift+Z — отмена / возврат
  • Ctrl+click по объекту в реестре — центрировать на схеме

Связи с другими модулями

  • Из tech-workspace — через handoff (создаёт engine.scheme.v1)
  • В scs-config — consumer-узлы со subtype=rack автоматически становятся стойками
  • В scs-design — те же стойки попадают в план зала
  • В mdc-config — для МЦОД-проектов: расстановка зон
