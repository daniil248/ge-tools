Назначение

Модуль «Технолог ЦОД» покрывает предпроектную стадию ЦОД: формирование концепции объекта (количество стоек, мощности, резервирование, площади, ТП/ДГУ) до углубления в детальное проектирование.

Что входит: количество и мощности стоек, IT-нагрузка, ИБП-системы, кондиционеры, резервирование, ТП/ДГУ, концептуальные требования к PDU, площади, multi-variant compare, пояснительная записка.

Что НЕ входит: СКС (отвечает СКС-инженер), активное/пассивное оборудование в стойках (заказчик/IT-отдел), детальный выбор моделей (электрик), кабельные журналы (электрик).

Структура UI

Два режима входа

  • 📋 Список — заполнение параметров по 5 секциям. Подходит когда есть готовое ТЗ.
  • 🗺 План — drag-drop помещений и оборудования на сетку (в разработке).
  • 📊 Сравнение — see статью.

Несколько групп стоек

В одной концепции может быть несколько групп стоек разной мощности (см. статью):

  • Серверные стойки 8×7 кВт (профиль IT-rack)
  • GPU-стойки 4×12 кВт (профиль GPU-heavy)
  • Сетевые стойки 2×3 кВт (профиль Network — НЕ входит в IT-нагрузку)

Аналогично для ИБП: разные системы для IT и для климата с разными покупательскими параметрами.

Привязка к каталогу

Каждая группа имеет кнопку 📦 Привязать модель… — открывает picker с моделями из общей библиотеки. После привязки параметры (widthMm/depthMm/demandKw/kva) автоматически копируются из выбранной модели.

Workflow

  1. Создайте вариант концепции (➕ слева).
  2. Заполните 5 секций (стойки → ИБП → климат → ввод → площади авто-считаются).
  3. (Опционально) привяжите модели через 📦.
  4. Создайте альтернативные варианты (📋 Дублировать), сравните в 📊 Сравнение.
  5. Выгрузите ПЗ через 📄 Пояснительная записка (HTML с возможностью «Печать → PDF»).
  6. Согласуйте с заказчиком, отметьте основной вариант (⭐).
  7. Нажмите 📤 Передать в проектирование — концепция превратится в узлы схемы (см. handoff).
⚠ После handoff'а variant становится read-only — повторно его редактировать нельзя. Можно создать новый вариант если нужно.
