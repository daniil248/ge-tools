Назначение

Компоновщик шкафа — содержимое каждого экземпляра шкафа: юниты (U), активное и пассивное оборудование.

Источники стоек

  • Из engine-схемы через scheme-rack-bridge.js: каждый consumer-rack узел со count=N разворачивается в N экземпляров.
  • Из POR-объектов проекта: raschet.project.<pid>.por.objects.v1 с типом rack.
  • Из rack-config.instances.v1 (legacy storage).

Tombstones

Удалённые стойки помечаются в raschet.project.<pid>.rack-config.tombstones.v1. При следующем bootstrap migrate-функция не воссоздаёт их из legacy storage. Сохраняется чистота данных.

Связи

  • scs-design — план зала с этими стойками
  • Реестр IT-оборудования — S/N, IP, MAC активного оборудования
